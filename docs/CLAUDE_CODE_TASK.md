# Claude Code 任务说明：机标结构化修改器 MVP

## 业务背景

第二轮任务是对第一轮「不符合」的机标内容进行细节修改。原始机标文本包含多层结构：

1. 一级：镜头，例如 `[镜头 1][0.00s-0.76s]`
2. 二级：说话单元，例如 `{"说话内容": "疼", ...}`
3. 三级：字段，例如说话内容、开始时间、结束时间、说话人、性别、是否人物可见

人工直接改纯文本很慢且容易破坏格式。本 MVP 目标是做一个独立、简单、清晰的结构化编辑器。

## 重要约束

1. 先做独立前端页面，不接 Label Studio。
2. 不要上复杂框架，当前为原生 HTML/CSS/JS。
3. 不要引入构建工具、npm、React、Vue，除非用户后续明确要求。
4. 前端必须直观清晰，方便业务人员理解。
5. 逻辑必须保持简单，不要混入第一轮 DOM Bridge 自动提交流程。
6. 当前输出只需要生成最终机标文本并支持复制。

## 数据结构

前端内部使用：

```js
{
  shots: [
    {
      id: "shot_xxx",
      shot_no: 1,
      shot_start: 0.00,
      shot_end: 0.76,
      collapsed: false,
      utterances: [
        {
          id: "utt_xxx",
          content: "疼",
          start: 0.02,
          end: 0.52,
          speaker: "speaker_0",
          gender: "男",
          visible: "不确定"
        }
      ]
    }
  ]
}
```

## MVP 必须保留的功能

1. 粘贴原始机标文本。
2. 点击「解析成模块」。
3. 一页展示所有镜头和说话单元。
4. 镜头卡片支持改时间、新增说话单元、插入镜头、删除镜头。
5. 说话单元卡片支持改说话内容、时间、speaker、性别、人物可见。
6. 说话单元支持复制、删除、上移、下移。
7. 输出格式自动生成，不让人工手写 JSON 标点。
8. 校验结果必须清晰显示。
9. 生成结果支持复制。

## 解析逻辑注意事项

原始机标可能不是严格 JSON，例如：

```text
"是否人物可见": 是
"是否人物可见": 否
```

所以不要直接 `JSON.parse`，当前代码使用字段正则提取。后续维护时要保持对非严格 JSON 的兼容。

## 输出格式

当前输出统一生成如下格式，字符串字段都加引号：

```text
[镜头 1][0.00s-0.76s]
{"说话内容": "疼", "开始时间": 0.02, "结束时间": 0.52, "说话人": "speaker_0", "性别": "男", "是否人物可见": "不确定"}
```
